# Step 0: Portal Setup — Branding & Identity

**Goal:** Make the portal feel like it belongs to Matar's Iced Americano. First impressions matter — a branded portal shows this isn't a generic tool, it's *their* platform.

## What We're Setting Up

Before building the data model, we'll customize the portal:

1. **Portal Name** — "Matar's Iced Americano Developer Portal"
2. **Logo** — Company logo in the sidebar
3. **Theme** (optional) — Colors that match the brand

## Why This Matters

> "This isn't Port. This is *our* internal developer portal that happens to be powered by Port."

Branding creates ownership. When engineers see their company's logo, they're more likely to treat it as a real tool, not a demo.

## How to Set Up

### Via Port UI

1. **Go to Settings** → Organization Settings
2. **Portal Name:**
   - Find "Portal Name" or "Organization Name"
   - Set to: `Matar's Iced Americano`
3. **Logo:**
   - Upload company logo (recommended: 200x50px PNG with transparent background)
   - Logo appears in the top-left of the sidebar
4. **Theme (optional):**
   - Primary color: Coffee brown `#6F4E37` or your brand color
   - Accent color: Cream `#FFFDD0` or complementary

### Logo Suggestions

For the workshop, you can use:
- A simple coffee cup icon
- Text logo "MIA" (Matar's Iced Americano)
- Any placeholder that looks professional

**Free logo resources:**
- [Logoipsum](https://logoipsum.com/) — placeholder logos
- [Flaticon](https://www.flaticon.com/) — coffee icons
- Or just use text: "☕ MIA"

## Demo Talking Point

When you show this in the workshop:

> "First thing we do is make this portal theirs. We're building for Matar's Iced Americano — a coffee delivery platform. Let's update the name and logo so it feels like home."

This takes 30 seconds but sets the tone: we're building something real, not playing with a demo.
